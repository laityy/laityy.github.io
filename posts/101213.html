模块化开发：

在开发中，随着ajax的出现和前后端的分离，客户端所需要完成的事越来越多，相对应的代码量也越来越多

为应对这种代码量的剧增，通常将代码保存在多个js文件中，但不同的文件之间会有函数或其他的依赖关系，而且还会有命名冲突问题，这时使用一定的逻辑关系将这些不同的文件串联起来，这一个个js文件就被称为一个个模块，组织起来就叫模块化，而这种开发方式就叫模块化开发

模块化开发通常需要遵守相应的规范，常见的模块化规范包括：CommonJS，ES6，AMD，CMD 等标准，但前两种相对使用较多，因此下面主要记录前两种模块化开发规范

CommonJs

​ 在nodejs中的使用的模块化开发方案，其运行通常依赖一些底层模块

1基本语法：

1.1module1.js 导出：

module.exports = {   //对象内包含需要被导出的变量和值
    flag: true,
    sum(a,b){
        return a+b;
    }
}

此时module1.js就是一个模块，模块内部也可以require其他模块

1.2在main.js中导入：

let moduleA = require('./module1.js') //声明一个变量接收，moduleA内保存的就是一个对象，然后访问里面的属性和方法即可

ES6模块化

1.基本语法

1.1导出模块：

export xxx

1.2导入模块：

import xxx from '路径'

2.项目案例：

2.1项目初始化

在project文件夹内创建如下文件

js
    | src //存放js源文件
        | module1.js
        | module2.js
        | module3.js
        | main.js
index.html

2.2代码编写

export有多种导出方式，下面在四个文件中分别演示：

(1)module1.js
//暴露模块：采用分别暴露的方式

export function foo1() {  
    console.log('我是 module1 中的 foo1');
}

export function foo2() {
    console.log('我是 module1 中的 foo2');
}

export let arr = [1, 2, 3, 4, 5];
(2) module2.js
//暴露模块：采用统一暴露的方式

function fn1() {
    console.log('我是 module2 中的 fn1');
}

function fn2() {
    console.log('我是 module2 中的 fn2');
}

//统一暴露
export { fn1, fn2 };

//使用export default方式导出
export default {
    name: 'wangs',
    sum(){
        console.log('module2.js文件里面的default')
    }
}

//导出一个基本数据类型
export let a = 'aaaaaaa'
(3)module3.js
//采用默认暴露的方式。
//默认暴露的方式可以暴露任意数据类型，暴露的是什么数据，接收到的就是什么数据

//语法格式：export default value;
export default {
    name: '张三',
    age: 12,
    sayHi: function () {
        console.log('Hello');
    }
}
(4)module4.js
//可以导出任意数据类型的值，string，number，boolean，null，undefined

var aaa = 'asdasd'; //aaa是字符串
// const aaa = 1231;  //数值
// let aaa = true; //布尔
// let aaa = null;
// let aaa = undefined;
export default aaa;
(5)main.js

主模块：使用import引入上面的文件

import { foo1, foo2 } from './module1.js'; //采用解构赋值的形式进行导入。注意，括号里的对象名，要和 module1 中的对象名一致。
import *as module2 from './module2.js';  //采用统一导入模式，使用 *as 语法，后面跟自定义对象名（module2），对象内包含目标js文件需要被导出的内容
import default2 from './module2.js'; //module2中用default方式导出的内容
import zhangsan from './module3.js'; //module3 模块是采用 default 方式进行暴露的，zhangsan这个名字随便起
import staticc from './module4.js'; //module4 模块同样是采用 default 方式进行暴露的，static名字随便起，这里表示一些静态值

foo1();
foo2();
module2.fn1();
module2.fn2();

console.log(module2); //module2对象  包含所有需要被打印的内容
console.log(default2); //输出module2文件export default导出的内容
console.log(module2.default); // 输出export default导出的内容  与 上面的default2全等

console.log(zhangsan); //输出张三对象
console.log(zhangsan.sayHi); //调用张三的方法
console.log(zhangsan.name); //调用张三的属性


console.log(staticc); //输出module4里面的基本数据类型
注意：
  • default在一个模块中只能使用一次,不允许存在多个
  • module1和module2中的统一暴露是采用常规暴露的形式，在引入它们时，模块名要一致。而且，要求用对象解构赋值的形式，而不是用 import myModule from ...这种形式（否则会报错 undefined）。
  • module2和module3是采用默认暴露的形式，在引入它们时，模块名随便起。
(6)index.html
<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>

<body>
    <script src="dist/main.js" type='module'></script>
</body>

</html>
